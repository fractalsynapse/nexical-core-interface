When we refer to a knowledge hub, we are describing a UI and API driven knowledge management system, not chatbot or database only systems These services are normally limited to customization through API integrations with other apps These services relative value and expense is determined by how they are used These services either implement teams at higher subscription tiers or for administrative use These services may occasionally provide a physical hosting option, such as AWS Outposts, usually at a premium price These services support a limited set of AI providers, and you can not add new ones There are a few services like Perpexity that seek to create public search engines with summarization, but most are wrapper chatbots or AI bolt-ons to existing products Some AI SaaS services may implement project based workflows, but many don't You can find someone to build you what you want, but many AI agencies are focused on wrapping a few services and AI providers into simple chatbots Most of these applications are open source and can be customized, and the ease of customization depends on the architecture Some of these applications can run a local server and could be made web accessible, but they are mainly made for personal use
